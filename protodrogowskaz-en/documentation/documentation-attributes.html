Select dropdowns
Basic attributes:
simply-select-dropdown="State param name"
Dropdown wrapper should have this attribute, which creates association with specifed state param
simply-select-dropdown-list
Add this attribute to element with list of options (the container that appears and disappears on click)
simply-select-dropdown-button
Add this attribute to element which is the button to show/hide the list of options
choice-value="State value to set"
Add this attribute to element from the list of options. Clicking it will set the param to this value
chosen-value="State value"
Elements with this attribute will be shown if param value matches the attribute value. Simply add this attribute to the elements inside dropdown button, that will show currently selected value.
If you want to have an empty placeholder, simply add another element that points to chosen-value="not-selected" and set default state value to "not-selected"
Dynamic CSS classes:
is-selected
This class is added to element in the list that is currently selected. Use it to show which element is already selected, for example show a check near this item.
UX information:
It is good to have a subtext explanation of an option native dropdowns do not provide this
It is good to have a different full explanation of the dropdown list, but only shorthand label in the chosen dropdown state
It is recommended that the empty placeholder has text "Select..." but you have full control over the empty state placeholder here
It is recommended to mark currently selected option in the dropdown list
Open dropdowns should not be too long - use max-height and overflow:auto to add inner scroll for the list
Advanced coding:
You can dynamically render the list of options by binding it to state param with array and using other regular list rendering techniques
The dropdown doesn't care about consistency of the style inside and HTML structure, you can do whatever you want inside
